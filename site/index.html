What memory is

Memory is the capture of the experience you and your agent live through as you work. It keeps the important decisions, the milestones, the successes and the failures — because all of it is part of the experience.

Neonmem carries that experience, and it holds a self-contained vector database with all the data your project needs. Neonmem is a format, and it is an application that improves the way you work with your agent.

A memory that stores experience.

Agents are stateless between sessions, so it's all about context — better context, better result. But what if the context is big? What if it outgrows you? What if your project is complex, with a lot of documents and a large, specific codebase? For your average agent it will be hard to help you without knowing these things. What if your current task chain is long, and to finish the last step you need to remember the first? Neonmem gives the agent an instrument to keep these things in mind — your decisions, the rationale behind them, the approaches you ruled out, and of course the failures. Over time it build expirience grid that allow you agnet to be more presise to exact you approaches and prpeferences.

The idea behind all of this is to augment both the person and the agent, not to replace. This tool is here to help you and your agent keep it all in mind — to share the same experience, to be on the same page.

Importer

Bootstrap with Neonmem

When you start, the memory is empty — but with the importer you can bootstrap memories from a project you've already worked on: reference material, and the decisions you made along the way with your agent. All of it can be imported into memory. The importer treats each correctly — your docs & code become a searchable pool, your agent chats become the decisions, dead-ends and rules worth keeping, and any file a chat references is pulled in too. memory cartridge

Neonmem is a single portable .neonmem file that carries everything the agent needs to be aligned with you and the project — the facts, the structure, and an embedded model trained on it — and it stays on your disk, under your control.

Load the cartridge and the agent gets the whole understanding at once: what the project is, how it's organised, and exactly where you left off.

A reasoning graph

Typed nodes and edges — observations, decisions, dead-ends, rules, procedures and plans — modelling the past, the present and the intended direction.

A comprehension layer

Neonmem clusters the memory into subsystems and distils a project skeleton and resume cursor — its structure and the current open thread. A new session receives the full picture, not a cold scroll-back.

An embedded model

A compact model, trained on your own memory, ranks what matters, senses tone and classifies what's captured. Self-contained and fully offline, it travels inside the cartridge.

In practice

Bring an existing project into memory

Consider an established project: a folder of design documents and API specifications, a set of meeting transcripts, and a codebase larger than any one person can hold in mind. Rather than re-explaining it each session, point the importer at all of it — in a single pass it becomes a connected memory: the decisions, the endpoints, the call graph and the plans. The agent opens already aware of the project.

Run it again whenever things move and the importer updates the memory in place — merging what's new, skipping what it already knows — so the cartridge stays current as the work does.

The actual tool: folders & files become a searchable pool, agent chats become the decisions and dead-ends worth keeping — and any file a chat references is pulled in too.

…and out comes a connected memory — the facts and the call graph, self-organised into subsystems, ready to hand straight to your agent.

Reads documents and notes

Markdown, text, PDF, Word and meeting transcripts — extracting the facts (endpoints, configuration, decisions) and the plans (roadmaps, goals, next steps), and discarding the prose.

Maps your code

Parses source into functions, classes and a call graph — what is defined where and what calls what — so the agent understands the structure of the codebase.

Creates and updates

Build a new memory, or re-run to update an existing one — new facts merged, duplicates skipped, secrets redacted, provenance retained. Fully local.

Built-in offline AI

An embedded model performs the comprehension — no cloud, no GPU, no API key. Runs on any modern CPU and ships inside the application; fully offline.

A searchable knowledge layer

Imports form a canonical, deduplicated, searchable facts layer with hybrid retrieval — and stale facts are pruned automatically, so the cartridge stays sharp as it grows.

One consolidated cartridge

Import your agent history (detected automatically), PDFs, documents and transcripts — the originals are retained, compressed and optionally encrypted, inside a single private store.

One pass, and a pile of docs, transcripts and code becomes an understanding-ready cartridge of your whole project.

FAQ

What is Neonmem?

A live, dynamic memory that makes your AI personal. It learns how you think and work — remembering decisions, dead-ends and reasoning across sessions — so your agent becomes a colleague who knows you. And you can watch it in 3D.

Does it only help with code?

No — coding, DevOps and config, and also creative writing (characters, plot, loose ends, rejected ideas).

Is my work private?

Yes — Neonmem runs locally with offline embeddings. No cloud calls, no API cost.

Which platforms?

Windows and Linux (AppImage) today. macOS is on the roadmap.

Where is my memory stored?

In a single local .neonmem file — a memory cartridge in Neonmem's own compact binary format. It carries the reasoning graph, quantized offline embeddings, the comprehension layer (subsystems + resume skeleton) and the small self-built model — all in one file. No database, no cloud: it stays on your machine and is the only thing your agent reads and writes.

What's a "memory cartridge"?

It's the whole mind in one portable file: the graph of what happened, the understanding of how it fits together, and a small model trained on your memory. Load it and your agent is instantly up to speed — back it up, move it between projects, keep it forever. It's yours.
