Writing journey

The Glass Lighthouse

Drafting a stage play.

Step 1 · Frame the story · OBSERVATION

The writer frames the play. Agent takes in the world. Neonmem stores the story's first thread.

Step 2 · Frame the story · OBSERVATION

The writer names the cast. Agent notes each one. Neonmem keeps the characters together.

Step 3 · Frame the story · QUESTION

The writer raises an open question. Agent holds it open. Neonmem records it for the debate.

Step 4 · Set the plan · PLAN

The writer sets the direction first. Agent records it as a PLAN — Neonmem's forward-looking node (indigo), held close while active so the through-line and next beats are never lost between sessions.

Step 5 · Design debate · HYPOTHESIS

The writer floats an idea. Agent treats it as a hunch. Neonmem files it as a hypothesis.

Step 6 · Design debate · DEBATE

The writer opens a debate on structure. Agent lays out both. Neonmem starts the debate thread.

Step 7 · Design debate · ARGUMENT

The writer weighs depth against clarity. Agent balances them. Neonmem links each argument to the debate.

Step 8 · Design debate · RESOLUTION

The writer decides. Agent commits to the structure. Neonmem stores the resolution for good.

Step 9 · Write it · ACTION

The writer asks for the scene. Agent drafts the beat. Neonmem logs the action and the planted setup.

Step 10 · Write it · CONCLUSION

The writer confirms it works. Agent validates the beat. Neonmem records the conclusion.

Step 11 · Hit a wall · DEAD_END

The writer flags a misfire. Agent marks it. Neonmem stores a permanent dead-end so the cut subplot doesn't creep back.

Step 12 · Hit a wall · RULE

The writer sets a craft rule. Agent adopts it. Neonmem pins it near the core.

Step 13 · Instant knowing · PROCEDURE

The writer records a repeatable method. Agent captures the steps. Neonmem stores the how-to.

Step 14 · Instant knowing · REFLEX

The writer sets the through-line. Agent internalizes it. Neonmem keeps it as a reflex.

Step 15 · Lean on memory · RECALL

The writer asks about a planted thread. Agent answers from memory. Neonmem surfaces the Act 1 setup and its payoff.

Step 16 · Lean on memory · DEAD_END_CHECK

The writer almost revives a cut idea. Agent warns it's a recorded dead-end. Neonmem flags why it was dropped.

Step 17 · Lean on memory · TIMELINE

The writer asks what happened today. Agent replays the beats in order. Neonmem returns them by time.

Step 18 · Let it sleep · DREAM

The writer wraps up. Agent consolidates. Neonmem merges the noise and keeps the characters, decisions and dead-ends.

Step 19 · Feed it the research · OBSERVATION

The writer feeds in historical research. Neonmem gathers the key facts — dates, roles, the storm of 1824 — and keeps them beside the story, skipping the filler.

Step 20 · Know the whole · SKELETON

The writer asks for the big picture. Agent reads it from Neonmem's comprehension layer — a small self-built model that clusters every node into subsystems. The brain colours by thread and Neonmem returns the play's skeleton, not a re-reading of the notes.

Step 21 · Know the whole · RESUME

The writer tests a cold start. Agent answers with the through-line, the cast and the resume cursor — the exact thread left open — without scrolling back. The glowing coherence core hands a fresh session the whole story at once.
