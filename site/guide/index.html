Give your AI agent experience

New here? This page explains — in plain language — what Neonmem is, how to install it, how to use it day to day, and how it actually remembers. No jargon required.

What is Neonmem, really?

Your AI assistant is brilliant, but it forgets. It has tools to remember context, but that's it — think of a genius with a really short memory.

Neonmem fixes that. It builds a private, local experience of your project — your docs, your code, the decisions you made and the dead-ends you hit — and feeds the right pieces to your AI exactly when it needs them. The result: an assistant that behaves like a colleague who actually has experience with your work, instead of a clever stranger you meet again every morning.

The cartridge — our whole philosophy in one idea

Cartridge

Your memory lives in a single file we call a cartridge (.neonmem). Think of it like a game cartridge or a memory stick: one portable thing that holds everything your agent has learned about a project.

  • It's yours. It sits on your machine. No cloud, no account, nothing phoning home.
  • It's portable. One file. Copy it, back it up, carry it to another machine, hand it to a teammate.
  • It's complete. It carries the knowledge and the understanding of how it fits together — not just a pile of text.

That's the core belief behind Neonmem: experience over raw knowledge — and experience you can keep, in one file.

Install — pick whichever suits you

① Windows — the app (easiest)

  1. Download the installer: Neonmem-Setup-0.9.8.exe
  2. Run it. If Windows shows a SmartScreen prompt, click More info → Run anyway (the build is signed).
  3. On the options screen, leave "Install the neonmem command-line tool" ticked if you also want to use Neonmem from a terminal — see ③ below.
  4. Finish, and launch Neonmem from the Start menu.

② Linux — the AppImage

  1. Download the AppImage: Neonmem-0.9.8-x86_64.AppImage
  2. Make it executable: chmod +x Neonmem-0.9.8-x86_64.AppImage
  3. Run it: ./Neonmem-0.9.8-x86_64.AppImage — no install, no root.

③ The command-line tool (for terminal lovers / no UI)

Prefer the keyboard, or running on a server with no desktop? Neonmem ships a command-line tool too — the same engine, no window.

  • Windows: tick the command-line option during install (step ③ above). Then open a new terminal and run neonmem --help.
  • Linux / Portable: it's included in the package — run the neonmem binary from the extracted folder.

Common commands: neonmem ingest <folder> to build memory from files, neonmem ingest-chat <transcript> to learn from an agent chat, and neonmem --help for the rest.

④ No-install — run without installing

Want zero installation — run it from a folder or a USB stick? Use the no-install build. See No-install →

Already built a memory and want to use it inside a web AI chat (ChatGPT / Claude on the web)? That's Portable memory — carry your project into any chat.

How to use it and your first five minutes

  1. Open Neonmem and create a project — this is where its cartridge lives.
  2. Feed it your files. In the importer, add your folders & files (docs, code, notes) and, optionally, an agent chat transcript. Click Build memory.
  3. Tag the topics that matter (optional) so they're surfaced first — e.g. API, deployment.
  4. Wait for "Done". You'll see two honest counters: Facts loaded and Memories created.
  5. Ask your AI about your project. Neonmem quietly hands it the relevant memory before each answer — so it responds from your sources, or honestly says it doesn't know.

What can it do?

  • Answer from your project. Ask "what does our internal term ARC mean?" and it tells you what your docs say — not a textbook guess.
  • Remember decisions & dead-ends. It keeps the choices you made and the approaches that failed, so your agent stops repeating mistakes.
  • Never invent. If something isn't in your memory, it says so instead of making it up.
  • Cover more than code. Coding, DevOps/config, even writing a book or screenplay , all this agent can help and neonemem helps agent

How it done or the memory layers

Neonmem inspiered of ike a brain, it keeps different kinds of memory in different places, so the right thing surfaces at the right time.

ReflexesThings it should always know instantly. Key definitions and standing rules — e.g. "ARC = our provisioning platform" or "never commit secrets". Surfaced first, every time.
Short-termRecent, working memory. Fresh observations and decisions you're actively dealing with.
Long-termDurable knowledge. Established facts, procedures, and dead-ends — the failed approaches kept as warnings so they're not repeated.
Facts poolEverything, searchable. Your whole imported corpus, vectorised and deduplicated — so any detail can be recalled on demand.
DreamsTidying up while idle. Like sleep, Neonmem consolidates duplicates and lets go of stale facts — so the memory stays sharp instead of becoming a junk drawer.

The big idea: a good memory is as much about forgetting the noise as storing the signal. That's why Neonmem keeps one clean, canonical statement per fact — not a wall of raw text.
